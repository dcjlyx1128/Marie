# Marie 需求文档(Requirements)

## 背景与目标
Marie 当前是一个"精致 demo":能用 AI 给文件分类/重命名,但存在三个硬伤——分类不一致、对大文件夹又慢又贵、缺少真实使用场景。本次迭代目标:**把 Marie 从 demo 升级为一个可靠、便宜、能自动运行的 AI 文件管家。**

## 产品定位
> 一个可靠、便宜、能自动运行的 AI 文件管家:规则兜底保证快和省,AI 只在必要时出手,分类稳定可预测,并能监听文件夹自动整理。

## 需求清单

### R1 — 固定分类体系(配置驱动)【高】
- 用户在配置文件中定义固定的分类体系(大类 + 可选子类)。
- AI 只能把文件**归入预设分类**,不能每次自由发明新文件夹。
- **验收**:同一文件多次运行,分类结果稳定一致;所有产出目录都来自配置。

### R2 — 规则优先 + AI 兜底【高】
- 先用扩展名/文件名规则处理"显而易见"的文件(mp3、dmg、zip…),零 API 成本。
- 仅对配置中标记需要 AI 的大类(如 文档、图片)中的文件,才调用 LLM 细分。
- **验收**:典型文件夹中 ≥80% 文件不触发 API;AI 调用次数显著下降。

### R3 — 结果缓存【中】
- 按文件内容 hash 缓存 AI 决策,持久化到本地。
- 重复文件、重复运行不再重复调用 API。
- **验收**:对同一文件夹连续运行两次,第二次 AI 调用次数为 0。

### R4 — watch 自动整理【高,杀手功能】
- `marie watch <folder>`:监听文件夹,新文件到达后自动归类。
- 防抖,忽略下载中的临时文件(`.crdownload`/`.part`/`.download` 等)。
- **验收**:向被监听文件夹放入一个文件,数秒内被自动归到正确分类,且有日志、可撤销。

### R5 — 信任与安全【高,贯穿全程】
- `organize` 默认 dry-run(预览);只移动不删除;保留 undo 日志;重名自动避让。
- 扫描时跳过已整理的分类目录,保证幂等(不会反复搬动)。
- AI 无法判断时归入 `fallback`(如 `其他/待分类`),绝不臆测。
- **验收**:重复运行不产生重复移动;任何一次整理都能 `marie undo` 还原。

## 非目标(本期不做)
- 云端同步 / 多设备。
- GUI 图形界面(保持 CLI)。
- 文件去重清理(仅识别,不自动删除)。
- 后台开机自启服务(watch 先做前台运行,自启留作后续)。

## 配置示例(marie.yaml)
```yaml
base: "."                    # 整理到哪个根目录
ai_fallback: true            # 模糊文件是否调用 AI
model: qwen-plus
vision_model: qwen-vl-plus

categories:
  视频:   { ext: [.mp4, .mov, .mkv] }
  音频:   { ext: [.mp3, .wav, .flac] }
  安装包: { ext: [.dmg, .pkg, .exe] }
  压缩包: { ext: [.zip, .rar, .7z] }
  代码:   { ext: [.py, .js, .ts, .go] }
  图片:
    ext: [.jpg, .png, .gif, .webp]
    ai: true
    subcategories: [截图, 照片, 设计稿]
  文档:
    ext: [.pdf, .docx, .txt, .md]
    ai: true
    subcategories: [财务/发票, 工作, 学习/论文, 合同]

fallback: 其他/待分类
```

## 配置查找规则
- 就近优先:先找当前目录 `./marie.yaml`,再找全局 `~/.marie/config.yaml`。
- 都没有时使用内置默认分类,并提示用户可 `marie init` 生成。
